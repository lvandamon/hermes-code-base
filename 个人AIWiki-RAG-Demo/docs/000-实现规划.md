# 个人 AI Wiki / RAG Demo 实现规划

## 一、阶段目标

### 阶段 0：代码库同步基础

目标：先把 `~/Documents/Hermes代码库/` 作为后续所有代码项目的统一根目录，并同步到 GitHub。

已采用方案：

```text
本地目录：~/Documents/Hermes代码库/
GitHub：git@github-code:lvandamon/hermes-code-base.git
同步脚本：~/.hermes/scripts/git-autosync.sh
自动频率：每 30 分钟
```

### 阶段 1：最小 RAG 文章处理工作流

目标：不接模型、不接向量库，只用固定 Prompt 让 Agent 生成稳定结构化笔记。

输入：一篇 Markdown 格式 RAG 文章。

输出：

1. 一页中文摘要
2. 术语中英文对照
3. 关键概念表
4. 关联主题
5. 下一步学习问题

### 阶段 2：Obsidian 化沉淀

目标：把 Agent 输出沉淀为可长期维护的 Markdown 知识笔记。

建议规则：

- 一篇文章一份笔记。
- 文件名优先使用：`YYYY-MM-DD-文章主题.md`。
- 英文术语首次出现必须中英文对照。
- 关键概念尽量用表格沉淀。
- 关联主题使用 Obsidian 双链或普通 Markdown 链接。

### 阶段 3：再考虑轻量 RAG

只有当前面流程稳定后，再考虑：

- Embedding（嵌入）：把 Markdown 切片转成向量。
- Retriever（检索器）：按问题召回相关笔记。
- Rerank（重排序）：对召回结果二次排序。
- Evaluation（评估）：检查摘要、术语、概念表是否稳定。

## 二、为什么先不做复杂架构

复杂架构容易导致早期精力分散：模型 API、向量库、UI、权限、部署、成本都可能成为阻塞点。

当前最重要的是：

1. 输入格式稳定。
2. Prompt 稳定。
3. 输出结构稳定。
4. 笔记可以长期复用。
5. GitHub 可以稳定同步。

## 三、完成标准

- 能从一篇 Markdown 文章生成一个完整 Prompt。
- Agent 按固定结构生成中文 Wiki 笔记。
- 笔记含中英文术语对照。
- 项目能提交并推送到 GitHub。
